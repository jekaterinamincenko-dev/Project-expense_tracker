# Izstrādes žurnāls

## 1. solis — plānošana
Izveidoju projekta plānu un datu struktūru. Sapratu,
ka labāk sadalīt programmu vairākos moduļos.

Problēmas: Grūti bija saprast ar kādu moduli sākt, un kā tos sasaistīt. Labi palīdzēja paraugs no 4. nedēļas.  

## 2. solis — pamata funkcijas
Izveidoju storage.py JSON saglabāšanai un app.py ar CLI.
Programma var pievienot un parādīt izdevumus.

Problēmas: Bija vairākas nezomīgas problēmas, kurus ilgi meģināju saprast, jo funkcijas nestrādāja, bet beigās izrādījas, ka aizmirsu aizvert iekavas, vai pēdiņas.  

## 3. solis — filtrēšana un analīze
Pievienoju filtrēšanu pēc mēneša, kopsavilkumu pa kategorijām
un izdevumu dzēšanu.

Problēmas: Pēc funkcijas 'delete_expense' izveidošanas, izdzēsti izdevumi joprojām tika radīti JSON failā. Šī problēma automātiski atrisinājās pārlādējot VSC.  

## 4. solis — CSV eksports
Pievienoju CSV eksportu ar csv moduli un izveidoju README
un dokumentāciju.

Problēma: Strādājot ar Git, sākumā nebija izveidots galvenais main branch, tāpēc radās kļūdas, mēģinot pārslēgties vai apvienot branch. Izveidoju main branch un pēc tam apvienoju feature branch, lai projekts būtu sakārtots un vieglāk pārvaldāms.
